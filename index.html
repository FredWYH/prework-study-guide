HTML

  • HTML code provides the frame or structure of the website or application.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS fill in or provides the style and colours of how the site or app will look.
  • An external CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • An * selector or wild card symbol declares ALL elements of the page will have the style properties written in the section
  • Width defined as 100% allows the stylesheet to adjust the content to fit to screen or device size.
  • Colour assignment can be either defined by semantic references like gray or blue, but more accurately defined by hexadecimal code or hex # like #ACACAC which can be found from https://htmlcolorcodes.com/

Git

  • Be as specific with terminal commands and git commands, for example, git pull only applies to the specific branch, whereas git pull origin main, will pull items across the main branch.
  • git status: checks what branch we are currently on
  • git diff: to find out what file difference are there to be committed.
  • git checkout -b branch-name: creates a new branch and switches to it.

JavaScript

  • JavaScript is the tool that takes a static HTML/CSS webpage and makes it interactive.
